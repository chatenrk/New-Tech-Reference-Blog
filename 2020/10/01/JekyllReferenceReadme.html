Pre-requisites

Jekyll requires the following:

  1. Ruby version 2.5.0 or higher
  2. RubyGems
  3. GCC and Make

See Requirements for guides and details

Instructions

  1. Install all prerequisites.
  2. Install the jekyll and bundler gems.
    gem install jekyll bundler
  3. Create a new Jekyll site at ./myblog
    jekyll new myblog
  4. Change into your new directory
    cd myblog
  5. Build the site and make it available on a local server
    bundle exec jekyll serve
  6. Browse to http://localhost:4000

Using Images in Jekyll

To use images in Jekyll pass the img src url as
{{site.baseurl}}/path/to/the/image/file

Adding Clean blog theme to Jekyll Site

  1. Replace the current theme in your Gemfile with gem
    "jekyll-theme-clean-blog".
  2. Run
    bundle install
  3. Replace the current theme in your _config.yml file with theme:
    jekyll-theme-clean-blog
  4. Build your site:
    bundle exec jekyll serve

Create gh-pages for unsupported themes

Depending if you deal with a User/Organization (UO) site or a Project site (P), do :

  1. From your working folder execute the following command
    git init
  2. Add a remote repository to your parent folder using the following command
    git remote add origin git@github.com:userName/userName.github.io.git (UO) or git remote add origin git@github.com:userName/repositoryName.git (P)
  3. Create a new code base using the following Jekyll command
    jekyll new
  4. In _config.yml, set the baseurl parameter to baseurl: '' (UO) or baseurl: '/repositoryName' (P)
  5. In gitignore add _site, as it would be versioned in the other branch
  6. Perform jekyll build to create the destination folder and build site.
  7. Checkout the repository
    git checkout -b sources (UO) or git checkout master (P)
  8. Add all the files for commiting
    git add -A
  9. Commit the code with a proper commit message
    git commit -m "jekyll base sources"
  10. Push the code to remote repository
    git push origin sources (UO) or git push origin master (P)
  11. Traverse to the _site directory
    cd _site
  12. Create an empty file called .nojekyll. This file tells gh-pages that there is no need to build
    touch .nojekyll
  13. Initialize the repo
    git init
  14. Add a remote repository to your _site folder using the following command
    git remote add origin git@github.com:userName/userName.github.io.git (UO) or git remote add origin git@github.com:userName/repositoryName.git (P)
  15. Checkout the relevant branch
    git checkout master (UO) or git checkout -b gh-pages (P)
  16. Add all files in the _site directory
    git add -A
  17. Commit the changes with proper commit message
    git commit -m "jekyll first build"
  18. Push to the remote repo, to gh-pages branch, which will be used to build the github pages site
    git push origin master (UO) or git push origin gh-pages (P)
